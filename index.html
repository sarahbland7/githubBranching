What is Github Flow?

"GitHub Flow is a lightweight, branch-based workflow that supports teams and projects where deployments are made regularly."

We're going to walk through the "Why?" and "How?" of this workflow.

What is a branch? Why use one?

At any given time in a project lifecycle, you're likely going to have multiple features or bugs to work on.

Some will be ready to go & some won't.

A "branch" is a "playground" version of the project... you get to explore without worrying that you'll damage the main project.

More on branches...

  • RULE: Anything in the master branch is deployable
  • NOTE: Your branch should be created off of master
  • TIP: Descriptive names are key!

How to use a branch?

  • Quick Github web interface demo...
  • Create a new branch on commmand line 
    
								git checkout -b <\new local branch name>
  • Switch to a different branch on command line 
    
								git checkout <\desired local branch name>
  • Check what branches currently exist (local & remote) 
    
								git branch -a

What next?

  • Add commits (using descriptive messages)

  • Open a pull request to initiate discussion about commits

  • Discuss + review code

  • Deploy from branch to run tests before merge

  • Merge code into master branch (more on this later...)
